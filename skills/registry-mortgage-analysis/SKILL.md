# 부동산등기부등본 근저당·담보 분석 스킬

## 개요

부동산등기부등본 PDF 폴더를 입력받아, 전체 파이프라인을 자동 수행한다:

1. **PDF 파싱**: 각 등기부등본에서 표제부·갑구·을구·하단 요약을 구조화
2. **근저당 추출**: 현행(말소되지 않은) 근저당의 순위번호, 설정일자, 접수정보, 채권최고액, 근저당권자, 채무자 추출
3. **공동담보목록 파싱**: 을구 본문에서 `공동담보목록 제YYYY-NNNN호` 형태의 정식 번호 추출
4. **순 담보금액 산출**: 공동담보목록 기준 그룹핑으로 중복 제거 → 실질 담보 한도 계산
5. **워크페이퍼 생성**: 4개 시트 구성의 감사용 엑셀 파일 산출

## 사전 준비

### 입력 요구사항

사용자에게 아래 정보를 확인한다:

| 항목 | 설명 | 예시 |
|------|------|------|
| PDF 폴더 경로 | 등기부등본 PDF가 들어있는 폴더 | `00_회사제시/05_유무형자산/등기부등본/` |
| 피감사회사명 | 워크페이퍼에 표기할 회사명 | `(주)삼성전자` |
| 감사기간 | 워크페이퍼에 표기할 기간 | `2025년 기말감사` |
| 결과 저장 경로 | 분석 결과를 저장할 폴더 | `99_Cluade 분석/부동산등기부등본/` |

PDF 파일 명명 규칙은 자유롭다. 스킬이 파일 내부의 고유번호를 자동 파싱한다.

## 파이프라인 상세

### Phase 1: PDF 파싱 — 부동산 기본정보 + 소유권

각 PDF에서 다음을 추출한다:

**표제부**:
- 고유번호 (예: `XXXX-XXXX-XXXXXX`)
- 유형 (토지/건물)
- 소재지, 도로명주소, 지목, 면적, 건물내역

**갑구 (소유권)**:
- 현재소유자, 등록번호, 취득일, 취득원인
- 공동소유 시 각 소유자의 이름, 지분
- 피감사회사 소유 여부 판정 (회사명에 피감사회사 관련 키워드 포함 여부)

**갑구 — 하단 요약 기준 가압류/압류 확인**:
- 하단 "주요 등기사항 요약"의 갑구 섹션에서 현행 가압류/압류 확인
- 본문 갑구의 취소선(말소) 항목을 현행으로 오인하지 않도록, 반드시 **하단 요약 기준**으로 판정

> 왜 하단 요약인가: 등기부등본 본문의 갑구/을구에는 말소된 항목도 취소선으로 표기되어 있다. PDF 텍스트 추출 시 취소선 정보가 유실되어 말소 항목을 현행으로 오분류하기 쉽다. 하단 "주요 등기사항 요약"에는 말소되지 않은 현행 사항만 기재되므로, 이를 기준으로 판정해야 정확하다.

### Phase 2: 을구 — 현행 근저당 추출

**하단 요약 을구 섹션** 기준으로 현행 근저당을 파싱한다:

```
순위번호 | 등기목적 | 접수정보 | 채권최고액 | 근저당권자
```

하단 요약에는 채무자가 누락되는 경우가 있으므로, **본문 을구**에서 순위번호별 채무자를 보충 파싱한다:

```python
# 본문 을구에서 순위번호별 채무자 추출 로직
# 1. 을구 섹션 시작점 찾기: 【을구】 또는 [을구]
# 2. 순위번호별 블록 분리: r'(\d+)\s+근저당권설정'
# 3. 각 블록에서 채무자 추출: r'채\s*무\s*자\s+(\S+)'
# 4. PDF 워터마크 텍스트("열", "람", "용") 필터링
```

### Phase 3: 공동담보목록 번호 추출

을구 본문에서 각 현행 순위번호에 해당하는 공동담보목록 번호를 추출한다.

```
공동담보목록 제2024-1155호
```

이 번호는 부동산등기법 제78조에 따른 공동담보의 법적 식별자다. 하나의 근저당이 여러 부동산에 걸쳐 설정될 때, 동일한 공동담보목록 번호를 공유한다.

**파싱 주의사항**:
- 말소된 순위번호의 공동담보목록과 현행 순위번호의 공동담보목록을 혼동하지 않도록, 순위번호별 범위를 격리하여 추출
- 담보추가(기존 근저당에 추가 부동산 편입)의 경우: 접수번호는 다르지만 공동담보목록 번호는 동일
- 정식 공동담보목록 번호가 없는 경우(inline 공동담보 또는 단독담보): 접수번호를 대체 키로 사용

### Phase 4: 순 담보금액 산출

공동담보목록 번호를 기준으로 그룹핑하여 중복을 제거한다. 핵심 로직:

```python
from collections import defaultdict

groups = defaultdict(lambda: {'amt': 0, 'items': [], 'key_type': ''})
for m in mortgage_data:
    # 1차: 공동담보목록 번호 (정식 식별자)
    key = m.get('공동담보목록', '')
    if key:
        key_type = '목록'
    else:
        # 2차: 접수번호 (폴백)
        key = 'receipt_' + m.get('접수번호', 'unknown')
        key_type = '접수' if m.get('공동담보') else '단독'

    if groups[key]['amt'] == 0:
        groups[key]['amt'] = m['채권최고액']
        groups[key]['key_type'] = key_type
    groups[key]['items'].append(m)

total_gross = sum(m['채권최고액'] for m in mortgage_data)
total_net = sum(g['amt'] for g in groups.values())
dedup_amount = total_gross - total_net
```

> 왜 이 방식인가: 동일 근저당이 10개 부동산에 공동담보로 설정되면 총 채권최고액은 ×10으로 과대계상된다. 공동담보목록 번호로 그룹핑하면 실질 담보 한도 ×1만 인정하여 정확한 순 담보금액을 산출할 수 있다.

### Phase 5: 워크페이퍼 생성

openpyxl로 4개 시트 구성의 엑셀 워크페이퍼를 생성한다.

#### Sheet 1: 근저당 상세

| 컬럼 | 내용 |
|------|------|
| No, 고유번호, 유형, 소재지 | 부동산 식별 |
| 소유자, 삼성전자소유, 지분 | 소유 현황 |
| 순위번호, 설정일자, 접수정보 | 근저당 기본 |
| 채권최고액 | 금액 (숫자 형식) |
| 근저당권자, 채무자 | 당사자 |
| 공동담보목록, 공동담보구분, 공동담보 | 공동담보 상세 |

- 모든 행에 부동산 정보를 빠짐없이 기재 (빈 셀 없음)
- 삼성전자 소유: 파란, 삼성전자 공동소유: 진파란, 제3자: 노란 배경

#### Sheet 2: 부동산 종합표

부동산별 1행으로 집계. 소유 현황, 근저당 건수/합계, 가압류 현황 포함.

#### Sheet 3: 요약

- 총 부동산 건수, 소유 구분별 건수
- 현행 근저당 건수, 채권최고액 합계
- 가압류 현존 여부
- 순 담보금액 (gross, 중복, net)
- 근저당권자별 집계
- 공동소유 부동산 상세
- 공동담보 그룹 분석 테이블

#### Sheet 4: 담보금액 분석

- Section A: 요약 (Gross → 중복제거 → Net)
- Section B: 공동담보 그룹별 순 담보금액 산출 내역 (금액 내림차순)
  - 그룹 식별자, 식별방식(목록/접수/단독), 근저당권자, 채무자
  - 건당 채권최고액, 대상물건수, Gross합계, 순담보금액, 중복제거액
  - 공동담보 그룹은 주황색 구분
- Section C: 공동담보 그룹 상세 전개 (2건 이상 그룹의 개별 물건)

## 실행 절차

스킬 실행 시 아래 순서로 진행한다:

1. **사용자 입력 확인**: PDF 폴더, 피감사회사명, 감사기간, 저장 경로
2. **PDF 목록 확인**: 폴더 내 PDF 개수, 파일명 출력
3. **Phase 1-2 실행**: 각 PDF에서 부동산 정보 + 현행 근저당 추출
4. **Phase 3 실행**: 공동담보목록 번호 정밀 파싱
5. **Phase 4 실행**: 하이브리드 그룹핑, 순 담보금액 산출
6. **중간 결과 보고**: 총 부동산 N건, 현행 근저당 N건, 순 담보금액 N원
7. **Phase 5 실행**: 4시트 워크페이퍼 생성
8. **최종 검증**: 빈 셀 0개, 합계 수식 검증, 공동담보 그룹 정합성

스크립트는 `scripts/` 폴더에 번들되어 있으며, 필요시 참조하여 수정·실행한다. 다만 스크립트는 범용 템플릿이므로 피감사회사별 특수 사항(소유 판정 키워드, 특이 등기 유형 등)은 실행 시 조정한다.

## 감사 관점 유의사항

- **가압류/압류**: 반드시 하단 요약 갑구 기준으로 판정. 본문 취소선 항목 오분류 주의.
- **공동담보목록 vs 접수번호**: 담보추가 케이스에서 접수번호만으로는 동일 근저당을 식별 불가. 공동담보목록이 법적 식별자.
- **채무자 확인**: 하단 요약에는 채무자가 누락될 수 있음. 본문 을구에서 보충 필요.
- **원본 수정 금지**: 회사제시 PDF 원본은 읽기 전용. 모든 결과는 별도 경로에 저장.

## 공동담보목록 파싱 한계 및 수동 검증 가이드

자동 파싱은 약 85~90% 정확도를 보인다. 아래 두 가지 유형에서 오류가 발생할 수 있으므로, 자동 파싱 후 반드시 수동 검증을 수행한다.

### 오류 유형 1: False Positive (말소 항목 오매핑)

을구 본문에서 말소(취소선)된 순위번호의 공동담보목록이 현행 순위번호에 잘못 매핑되는 경우. PDF 텍스트 추출 시 취소선 정보가 유실되기 때문이다.

**검증 방법**: 자동 파싱된 공동담보목록 번호가 해당 순위번호의 접수일자와 시간적으로 일관되는지 확인. 예를 들어 2020년 설정 근저당에 2025년 공동담보목록이 붙어 있으면 의심.

### 오류 유형 2: False Negative (복잡 PDF 누락)

근저당 10건 이상의 복잡한 등기부등본에서 순위번호별 블록 격리가 실패하여 현행 항목의 공동담보목록을 놓치는 경우.

**검증 방법**: 근저당 건수가 많은 부동산(5건 이상)은 원본 PDF를 열어 하단 요약의 공동담보 참조 여부를 확인. 해당 순위번호 근처에 `공동담보목록 제YYYY-NNNN호` 텍스트가 있는지 확인.

### 수동 보정 절차

1. 파싱 결과 JSON을 열어 `공동담보목록` 필드를 확인
2. False Positive 의심 건: `공동담보목록` 값을 빈 문자열로 변경
3. False Negative 의심 건: 원본 PDF에서 공동담보목록 번호를 확인하여 `공동담보목록 제YYYY-NNNN호` 형식으로 직접 입력
4. 보정 후 워크페이퍼 재생성
