## 1. DOUZONE

### 스마트A 계열 (가장 흔함)
```
승인일 | 승인번호 | 차변금액 | 차변계정과목 | 대변계정과목 | 대변금액 | 적요 |
거래처코드 | 거래처명 | 작성일 | 작성번호 | 프로젝트코드 | 프로젝트명 |
작성부서코드 | 작성부서명 | 사용부서코드 | 사용부서명 | ...
```

특징:
- 차변/대변이 한 행에 동시 표기 (차대통합형)
- 계정과목에 코드 내장: `[1350000]부가세대급금`
- 후반부에 전표유형, 사업장, 사원 등 추가 관리항목
- 별도의 계정과목코드/계정과목명 컬럼이 마지막에 존재하기도 함
- 확장 버전은 90열 이상 (사용자정의 필드 L1~L9, M1~M9 포함)

### iCUBE 계열
```
월 | 일자 | 전표번호 | 구분 | 코드 | 계정과목 | 차변 | 대변 |
거래처 | 적요 | 지출성격 | 부서·사원 | 신용카드 | 현장 | 프로젝트 | ...
```

특징:
- 차대분리형: '구분' 컬럼이 '차변' 또는 '대변'
- 한 행에 차변 또는 대변 중 하나만 기록
- 계정코드가 별도 컬럼
- 관리항목(신용카드, 현장 등) 각각 코드+명칭 쌍으로 구성

## 2. SAP

### SAP 표준 출력
```
Document Number | Posting Date | Document Date | Reference |
Account | Account Description | Debit Amount | Credit Amount |
Amount in Local Currency | Tax Code | Cost Center | Profit Center | ...
```

특징:
- 영문 헤더가 기본
- Document Number가 전표번호
- Amount in Local Currency가 원화 환산액
- Company Code, Business Area 등 조직 컬럼 다수

## 3. Oracle

### EBS Journal Export
```
Period | Source | Category | Batch Name | Journal Name | Line |
Account | Debit | Credit | Description | Reference |
```

특징:
- Period가 'JAN-25' 형식
- Account가 세그먼트 구조 (예: 01-1234-5678-00)
- Batch/Journal 계층 구조

## 4. ecount

### 이카운트 표준
```
일자 | 전표번호 | 구분 | 계정코드 | 계정명 | 차변 | 대변 |
거래처 | 적요 | 부서 | 프로젝트
```

특징:
- iCUBE와 유사한 차대분리형
- 비교적 단순한 구조

## 5. 수동 엑셀 분개장

### 2행 헤더형
```
Row 0: [전표승인정보] [     ] [차변] [     ] [대변] [     ] [적요] [거래처] ...
Row 1: [년/월/일    ] [번호 ] [금액] [계정  ] [계정] [금액  ] [    ] [코드  ] ...
```

특징:
- Row 0이 그룹 헤더 (merged cells)
- Row 1이 상세 컬럼명
- 날짜가 년/월/일로 분리
- 금액이 콤마 포함 문자열

### SUBTOTAL 선행형
```
Row 0: [None] [None] [SUBTOTAL] [271,286,389,734] [None] [SUBTOTAL] [271,286,389,734] ...
Row 1: [작성일] [승인일] [승인번호] [차변금액] [차변계정과목] [대변계정과목] [대변금액] ...
Row 2: [데이터 시작]
```

특징:
- 첫 행이 합계(SUBTOTAL)
- 실제 헤더는 두 번째 행
- 이후 일반 데이터

## 6. 실전 사례: 예시 회사 (2021~2025)

5개 연도에 걸쳐 3가지 서로 다른 구조:

| 연도 | ERP 추정 | 패턴 | 행수 | 열수 |
|------|----------|------|------|------|
| 2021 | iCUBE | 차대분리형 + 단일행 헤더 | 37,646 | 21 |
| 2022 | 수동 엑셀 | 2행 헤더 + 콤마 문자열 | 52,297 | 17 |
| 2023 | 스마트A 확장 | SUBTOTAL 선행 + 93열 | 83,434 | 93 |
| 2024 | 스마트A | 차대통합형 + 코드내장 | 96,305 | 30 |
| 2025 | 스마트A | 2024와 동일 | 90,103 | 30 |

이 사례에서 배운 핵심 교훈:
- 같은 회사라도 ERP 변경으로 연도별 구조가 완전히 다를 수 있음
- 2022년처럼 금액이 문자열인 경우 safe_num() 변환 필수
- 2023년처럼 SUBTOTAL 행이 선행하는 경우 헤더 인식 로직 필요
- 손상된 파일(zip 오류)은 원본 경로에서 재시도하면 열리는 경우도 있음
