# 한국 감사보고서 표준 패턴 (Korean Audit Report Patterns)

이 문서는 한국 일반기업회계기준(K-GAAP) 및 K-IFRS 감사보고서/재무제표 엑셀 파일에서 자주 보이는 시트 구조, 헤더 패턴, 소계 위치, 부호 처리 규칙을 정리한 참조 자료다.

## 1. 시트명 컨벤션

### 재무제표 시트

| 표준 시트명 | 한글명 | 변형 시트명 | 헤더 키워드 |
|---|---|---|---|
| BS | 재무상태표 | B/S, BalanceSheet, 재상표 | "재 무 상 태 표", "Balance Sheet" |
| PL | 손익계산서 | P/L, IS, IncomeStatement, 손계 | "손 익 계 산 서", "Income Statement" |
| CIS | 포괄손익계산서 | CI, ComprehensiveIncome | "포 괄 손 익 계 산 서" (별도 시트인 경우) |
| CE | 자본변동표 | SCE, StatementOfChanges | "자 본 변 동 표" |
| CF | 현금흐름표 | SCF, CashFlow | "현 금 흐 름 표" |

### 주석 시트

가장 흔한 컨벤션:
- **숫자만**: 시트명이 `1`, `2`, `3`, ..., `N` (DSD Smart Reviewer 형식)
- **주석N**: `주석1`, `주석2`, ...
- **N. 제목**: `1. 회사의 개요`, `4. 유형자산` (보기 좋지만 시트 참조 시 따옴표 필요)

### 표준 주석 순서 (K-GAAP 일반)

| 번호 | 주제 |
|---|---|
| 1 | 회사의 개요 (자본금/주주현황) |
| 2 | 재무제표 작성기준 및 유의적 회계정책 |
| 3 | 매출채권 등의 양도 및 할인 |
| 4 | 유형자산 |
| 5 | 무형자산 |
| 6 | 보험가입자산 |
| 7 | 지분법적용투자주식 |
| 8 | 매도가능증권 / 단기매매증권 |
| 9 | 차입금 |
| 10 | 퇴직급여 |
| 11 | 법인세 |
| 12 | 우발채무 및 약정사항 (담보 제공) |
| 13 | 자본금 및 자본잉여금 |
| 14 | 이익잉여금 (이익잉여금처분계산서) |
| 15 | 포괄손익계산서 (별도 표시인 경우) |
| 16 | 특수관계자와의 거래 |
| 17 | 부가가치의 내역 |
| 18 | 현금흐름표 (현금유출입 없는 거래) |
| 19 | 금융부채의 유동성 위험관리 |
| 20 | 재무제표 확정일과 기관 |
| 21 | 보고기간 후 사건 |

**주의**: K-IFRS는 주석 번호와 주제 매핑이 다를 수 있다. 회사마다 다르므로 하드코딩 금지. **반드시 시트 본문의 "N. 제목" 패턴에서 동적으로 추출**한다.

## 2. 표준 헤더 패턴

### 회계연도 표기

```
제 7(당) 기 2025년 12월 31일 현재         ← BS (시점)
제 7(당) 기 2025년 01월 01일부터 2025년 12월 31일까지   ← PL/CE/CF (기간)
```

추출 정규식 예시:
```python
import re
period_pattern = re.compile(r'제\s*(\d+)\((당|전)\)\s*기\s*(\d{4})년')
# 매칭 그룹: (기수, 당/전, 연도)
```

### 단위 표시

`(단위: 원)`, `(단위: 천원)`, `(단위: 백만원)` 중 하나. 보통 C8 또는 H6 셀.

**중요**: 모든 시트가 같은 단위인지 확인. 주석에서만 천원 단위인 경우가 있어 교차검증 시 단위 변환 필요.

## 3. BS 표준 구조

### 컬럼 매핑

| 열 | 용도 |
|---|---|
| B | 과목명 |
| C | 당기 개별금액 |
| D | 당기 소계/합계 |
| E | 전기 개별금액 |
| F | 전기 소계/합계 |

### 소계/합계 행 패턴 (B열 텍스트)

| 패턴 | 예시 | 수준 |
|---|---|---|
| `I.`, `II.`, ... | "I. 유동자산" | 대분류 |
| `(1)`, `(2)`, ... | "(1) 당좌자산" | 중분류(소계) |
| 들여쓴 공백 + 명사 | "  대손충당금" | 차감 항목 |
| `자   산   총   계` | 띄어쓰기 강조 | 자산총계 |
| `부   채   총   계` | | 부채총계 |
| `자   본   총   계` | | 자본총계 |
| `부채및자본총계` | | 대차균형 |

### 검증 식 표준 형태 (BS 당기)

```
당좌자산(D12) = SUM(C13:C23)
재고자산(D24) = SUM(C25:C31)
유동자산(D11) = D12 + D24
투자자산(D33) = SUM(C34:C38)
유형자산(D39) = SUM(C40:C58)
무형자산(D59) = SUM(C60:C63)
기타비유동(D64) = SUM(C65:C67)
비유동자산(D32) = D33 + D39 + D59 + D64
자산총계(D68) = D11 + D32
유동부채(D70) = SUM(C71:C78)
비유동부채(D79) = SUM(C80:C82)
부채총계(D83) = D70 + D79
자본총계(D96) = 자본금 + 자본잉여금 + 자본조정 + 기타포괄 + 이익잉여금
부채및자본(D97) = D83 + D96
대차균형: D68 = D97  ← 가장 중요
```

## 4. PL 표준 구조

### 검증 식 표준 형태

```
매출액(D10) = C11
매출원가 구성(C13) = C14 + C15 + C16   ← 기초제품재고 + 당기제조 - 기말제품재고
매출원가(D12) = C13
매출총이익(D17) = D10 - D12
판관비(D18) = SUM(C19:C41)
영업이익(D42) = D17 - D18
영업외수익(D43) = SUM(C44:C52)
영업외비용(D53) = SUM(C54:C62)
법인세전순이익(D63) = D42 + D43 - D53
당기순이익(D65) = D63 - D64(법인세비용)
```

**부호 주의**: PL 매출원가의 "기말제품재고액"은 차감(음수) 표시. 따라서 `C13 = C14 + C15 + C16`에서 C16은 이미 음수.

## 5. CE (자본변동표) 표준 구조

### 컬럼 매핑

| 열 | 용도 |
|---|---|
| B | 과목/일자 |
| C | 자본금 |
| D | 자본잉여금 |
| E | 자본조정 |
| F | 기타포괄손익누계액 |
| G | 이익잉여금 |
| H | 총계 (행합계) |

### 검증 형태

- **행 검증**: `H{row} = SUM(C{row}:G{row})`
- **열 검증**: 각 열에 대해 `전기말 = 전기초 + 변동들의 합`, `당기말 = 당기초 + 변동들의 합`
- 예: `C15(전기말 자본금) = C10(전기초) + C11(유상증자) + C12(지분법자본변동) + C13(자기주식) + C14(당기순이익)`

## 6. CF 표준 구조

### 부호 처리가 가장 복잡

CF는 활동별로 유입/유출을 나눠 표시하며, **유출 소계는 음수, 하위 항목은 양수**로 표시된다.

```
III. 영업활동현금흐름(D10) = C11(순이익) + C12(가산) + C22(차감) + C28(자산부채변동)
   1. 당기순이익(C11): 부호 그대로
   2. 비용가산(C12): C12 = SUM(C13:C21)            ← 양수
   3. 수익차감(C22): C22 = -SUM(C23:C27)            ← 부호반대 → 검증식: C22 + SUM(C23:C27) = 0
   4. 자산부채변동(C28): 증가는 음수, 감소는 양수가 일반적

IV. 투자활동현금흐름(D46)
   1. 투자유입액(C47) = SUM(C48:C56)               ← 양수
   2. 투자유출액(C57) = -SUM(C58:C74)              ← 부호반대 → 검증식: C57 + SUM(C58:C74) = 0

V. 재무활동현금흐름(D75)
   1. 재무유입액(C76) = SUM(C77:C78)               ← 양수
   2. 재무유출액(C79) = -SUM(C80:C82)              ← 부호반대

VI. 현금증감(D83) = D10 + D46 + D75
VII. 기말현금(D85) = D83 + D84(기초현금)
```

**검증식 작성 규칙**: 
- 표시된 합계가 음수이면서 하위 항목이 양수인 소계 → `=합계 + SUM(하위)` (덧셈)
- 표시된 합계가 양수이면서 하위 항목도 양수 → `=합계 - SUM(하위)` (뺄셈)

## 7. 차감(부호반대) 항목 식별 키워드

다음 키워드가 B열 과목명에 포함되면 음수(차감) 항목이다.

### BS 차감
- "감가상각누계액" (유형자산 차감)
- "대손충당금" (매출채권/대여금/미수금 차감)
- "평가충당금" (재고자산 차감)
- "국고보조금" (유형자산 차감)
- "퇴직연금운용자산" (퇴직급여충당부채 차감, BS상 음수로 표시)

### PL 차감
- "기말제품재고액" (매출원가 산출 시)
- "환입" 항목 (대손상각비 환입 등은 영업외수익이지만 음수)

### CF 차감 소계
- "현금유출이 없는 비용 등의 가산" → 양수
- "현금유입이 없는 수익 등의 차감" → **음수** (소계 자체)
- "투자활동으로 인한 현금유출액" → **음수** (소계 자체)
- "재무활동으로 인한 현금유출액" → **음수** (소계 자체)

## 8. 주석 시트 표 구조 패턴

### 헤더 행 식별

다음 키워드가 첫 셀에 있으면 헤더 행:
- "구  분", "구분", "과   목", "과목"
- "회사명", "거래처명", "차입처"
- "기초", "기말" 등 컬럼 헤더

### 합계 행 식별

B열 텍스트가 다음과 일치하면 합계 행:
- "합   계", "합계", "합 계"
- "소  계", "소계"
- "총  계", "총계"

### 행/열 모두 검증 가능한 표

전형적인 변동표(유형자산, 무형자산, 퇴직급여, 자본금 변동 등):
```
구분  | 기초 | 취득 | 처분 | 상각 | 기말
A항목 | ... | ... | ... | ... | ...
B항목 | ... | ... | ... | ... | ...
합계  | ... | ... | ... | ... | ...
        ^                          ^
        열검증: 합계행 = SUM(데이터행)
        
        행검증: 기말 = 기초 + 취득 + 처분 + 상각
```

## 10. 시트 참조 시 따옴표 규칙

엑셀에서 시트명이 **숫자만**이거나 **공백/특수문자 포함**이면 작은따옴표 필수.

```excel
=BS!D11             ← OK (영문)
=재무상태표!D11      ← OK (한글, 단일 단어)
='9'!C46            ← 필수 (숫자 시트명)
='주석 9'!C46       ← 필수 (공백 포함)
```

Python에서 동적 생성 시:
```python
def sheet_ref(sheet_name, cell):
    if sheet_name.isdigit() or ' ' in sheet_name or '-' in sheet_name:
        return f"'{sheet_name}'!{cell}"
    return f"{sheet_name}!{cell}"
```

## 11. 단위 환산 규칙

### 천원 단위 반올림 차이 허용 범위

서술에 "752,426천원"으로 기재되었을 때, 실제 표 합계가 752,425,789원이면:
- 천원으로 환산: 752,425.789 → 반올림 752,426 ✓ 일치
- 차이 허용: ±999원 (반올림 단위)

### 백만원 단위

"3,000백만원" → 3,000,000,000원. 차이 허용: ±999,999원.

## 12. 회사 정보 추출 패턴

| 정보 | 위치 (전형적) |
|---|---|
| 회사명 | BS B8, PL B8 (예: "주식회사 삼성전자") |

## 13. 자주 보이는 비표준 형식

자동 탐지가 실패할 수 있는 케이스:

1. **시트명이 한글 + 영문 혼합**: "재무상태표(BS)" 등
2. **소계 열이 D가 아닌 경우**: BS의 (당기) 컬럼이 C, D, E (개별/소계/합계 3분할)
3. **주석 번호가 시트명에 없는 경우**: 시트명 "유형자산", "차입금" 등
4. **합본 시트**: 모든 재무제표가 한 시트에 있는 경우
5. **Pivot/계산이 다른 시트에 분산**: 동일 항목이 여러 시트에 중복 표시

이런 경우 사용자에게 시트 매핑을 직접 확인해야 한다.
