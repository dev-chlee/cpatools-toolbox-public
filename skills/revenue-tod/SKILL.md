# 매출 Test of Details (TOD) 자동화 스킬

## 개요

외부감사 시 매출 샘플에 대한 Test of Details를 자동 수행한다.
샘플링 엑셀과 증빙 PDF 폴더를 입력받아, 각 증빙을 분류하고 텍스트를 추출한 뒤, 샘플 정보와 교차대사하여 상세 결과를 엑셀 워크페이퍼로 출력한다.

## 워크플로우

### Phase 1: 환경 파악

1. **폴더 구조 탐색**: 사용자가 지정한 폴더를 탐색하여 샘플링 엑셀 파일과 증빙 하위폴더 구조를 파악한다.
2. **샘플링 엑셀 읽기**: openpyxl로 엑셀을 열어 매출 유형별(국내매출/중계무역/직수출 등) 샘플 목록을 추출한다.
   - 엑셀의 시트명, 행 범위, 열 위치는 파일마다 다를 수 있으므로 사용자에게 확인하거나, 헤더 행을 자동 탐지한다.
   - 핵심 정보: 폴더명, 매출유형, CI번호, 거래처명, 운임조건(Incoterms), 통화, 외화금액, 원화금액
3. **증빙 파일 목록 작성**: 각 샘플 폴더 내 파일(PDF, 이미지 등)을 리스트업한다.

### Phase 2: 증빙 분류 (File Classification)

파일명 기반으로 증빙 유형을 분류한다. 분류 우선순위가 중요하다.

**분류 순서 (우선순위 높은 것부터):**

1. **세금계산서**: '세금계산서' 포함
2. **거래명세서**: '거래명세', '인수증' 포함
3. **POD (Proof of Delivery)**: 'pod', 'proof of delivery', 'delivery receipt', '배송완료', 'delivery order' 등
4. **BL/AWB (선적서류)**: 'bl', 'b/l', 'hawb', 'awb', 'waybill', 'fedex', 'dhl', '선적', '출고증', '출고관련서류' 등
   - 단, 'pod'가 포함된 파일은 제외 (POD로 분류됨)
5. **CI (Commercial Invoice)**: 'ci-', 'ci_', 'commercial invoice' 등
   - 단, 'cipl', 'ci&pl' 포함 시 INV/PL로 분류
6. **PO (Purchase Order)**: 'po ', 'po-', 'purchase order', 'p_us_', 'pi -' 등
   - 단, 'pod', 'invoice', '출고' 포함 시 제외
7. **INV/PL (Invoice/Packing List)**: 'inv_', 'invoice', 'cipl', 'ci&pl', '출고완료' 등
8. **OTHER**: 위에 해당하지 않는 파일

**주의**: '출고'라는 단어가 세금계산서 파일명에 포함될 수 있다 (예: "세금계산서 - 2025.03.24 출고.pdf"). 세금계산서를 INV/PL보다 먼저 체크해야 오분류를 방지할 수 있다.

### Phase 3: PDF 텍스트 추출

pdfplumber를 사용하여 각 PDF의 텍스트를 추출한다.

```python
import pdfplumber

def extract_pdf_text(filepath):
    try:
        with pdfplumber.open(filepath) as pdf:
            text = ""
            for page in pdf.pages:
                t = page.extract_text()
                if t:
                    text += t + "\n"
            return text.strip()
    except Exception as e:
        return f"[PDF READ ERROR: {e}]"
```

- 이미지형 PDF(스캔본)는 텍스트 추출 불가 → "[텍스트 추출 불가]"로 기록
- 이미지 파일(jpg, png)도 동일하게 텍스트 추출 불가로 기록
- 추출 불가 시 "수동확인 필요"로 표기. 절대 추정하지 않는다.

### Phase 4: 해외매출 테스트 (6개 항목)

각 해외매출(중계무역/직수출) 샘플에 대해 아래 6개 테스트를 수행한다.
**모든 테스트는 구체적으로 "무엇을 대사하여 어떤 결과를 얻었는지" 기록해야 한다.**

#### T1. PO (Purchase Order) - 고객 주문 확인
- PO 파일 존재 여부 확인
- PO 텍스트에서 거래처명 매칭 시도
- 기록 형식: `"PO 파일 확인: [파일명] / PO에서 거래처명 '[이름]' 확인"`
- PO 미확인 시: `"PO 서류 미확인 - 고객 주문 증빙 없음"` → Fail

#### T2. Invoice(CI) - 빌링 확인
- CI 또는 INV/PL 파일에서 텍스트 추출
- CI번호 대조: 엑셀상 CI번호가 Invoice 텍스트 또는 파일명에 존재하는지
- 거래처명 대조: Invoice 텍스트에 엑셀상 거래처명이 존재하는지
- 기록 형식: `"Invoice 파일: [파일명] / CI번호 '[번호]' → Invoice 상 확인됨 [일치] / 거래처 '[이름]' → Invoice 상 확인됨 [일치]"`
- Invoice 미확인 시: Fail

#### T3. 선적서류(BL/AWB/출고증) - 인도 확인
- BL, AWB, 출고증 등 선적/출고서류 존재 여부 확인
- 가능한 경우 선적일/출고일 추출
- EXW 조건: BL 없이 출고증이나 INV/PL로 대체 가능
- 선적서류 미확인 시: Fail

#### T4. Incoterms - 매출인식조건 충족 여부
- 엑셀상 운임조건과 Invoice상 Incoterms 대조
- Incoterms별 매출인식 기준:
  - **EXW, FCA, FOB, FAS**: 출고/선적 시점 → 출고증/BL 확인으로 충족
  - **CFR, CIF, CIP, CPT**: 선적 시점 → BL/AWB 확인으로 충족
  - **DAP, DDP, DAT, DPU**: 도착지 인도 시점 → POD 확인 필요
- Incoterms 불일치 시: Exception으로 표기하고 수동확인 필요 기록

#### T5. POD - 도착 확인 (D조건 해당 시)
- DAP/DDP/DAT/DPU 조건인 경우에만 적용
- POD 파일 존재 및 배송완료일 추출
- D조건인데 POD 미확인 시: Exception
- D조건이 아닌 경우: N/A

#### T6. 금액 대조
- Invoice 텍스트에서 숫자를 추출하여 엑셀상 외화금액과 대조
- tolerance ±1 이내 일치 시 Pass
- 외화금액 매칭 실패 시 원화금액으로 재시도
- 기록 형식: `"엑셀상 금액: USD 1,234.56 (KRW 1,650,000) / Invoice상 금액: USD 1,234.56 → 엑셀 USD 1,234.56 [일치]"`

### Phase 5: 국내매출 테스트

국내매출 샘플에 대해 아래 3개 테스트를 수행한다.

#### ① 세금계산서 확인
- 파일 존재 여부
- 공급가액 대조: 엑셀상 매출액(대변) vs 세금계산서상 공급가액
- 거래처 대조: 엑셀상 거래처 vs 세금계산서상 공급받는자
  - '(주)', '주식회사' 등 법인 표기 변형도 고려

#### ② 거래명세서(인수증) 확인
- 파일 존재 여부
- 합계금액 대조

#### ③ 금액 종합 대조
- 엑셀 대변금액 vs 세금계산서 공급가액 vs 거래명세서 합계

### Phase 6: 종합 판정

| 판정 | 기준 |
|------|------|
| **Pass** | 모든 핵심 증빙 확인 + 자동 대조 성공 |
| **Exception** | 증빙은 존재하나 자동대조 불가 (수동확인 필요) |
| **Fail** | 핵심 증빙(Invoice, BL 등) 부재 |

### Phase 7: 엑셀 워크페이퍼 생성

openpyxl을 사용하여 3개 시트로 구성된 워크페이퍼를 생성한다.

#### Sheet 1: Summary
- 회사명, 감사대상기간, 작성일
- 유형별 Pass/Exception/Fail 집계표
- Exception/Fail 목록 (폴더명, 유형, 결과, 사유, 관련 테스트)
- 테스트 절차 설명

#### Sheet 2: 해외매출 상세
- 샘플 기본정보 (폴더명, 유형, CI No., 거래처, 운임조건, 통화, 금액)
- T1~T6 각각의 **결과(Pass/Exception/Fail/N/A)**와 **상세 대사내역**을 별도 열에 기재
- 종합결과, Exception 사유, 증빙파일 목록

#### Sheet 3: 국내매출 상세
- 샘플 기본정보 (폴더명, 거래처, 공급가액)
- 세금계산서/거래명세서/금액대조 각각의 결과와 상세내역

#### 서식
- 헤더: 진한 파랑 배경 + 흰색 볼드
- Pass: 연두색 배경 / Exception: 노란색 배경 / Fail: 빨간색 배경 / N/A: 연파랑 배경
- 모든 셀: 얇은 테두리, 텍스트 줄바꿈
- 금액: #,##0 또는 #,##0.00 형식

## 핵심 원칙

1. **할루시네이션 금지**: PDF에서 읽지 못한 내용을 추정하지 않는다. "자동대조 불가 - 수동확인 필요"로 정직하게 기록한다.
2. **구체적 기재**: "일치" 또는 "불일치"만 쓰지 않고, 무엇(어떤 값)을 어디(어떤 서류)에서 확인하여 어떤 결과를 얻었는지 기재한다.
3. **증빙 기반**: 모든 판단은 실제 추출된 텍스트에 근거한다.
4. **분류 우선순위 준수**: 세금계산서 → 거래명세서 → POD → BL → CI → PO → INV/PL 순서로 분류한다.
5. **유연한 매칭**: 거래처명은 부분매칭, CI번호는 파일명도 함께 확인한다.

## 추가 권고 테스트 항목 (선택)

TOD 수행 후 아래 항목을 추가 검토하면 감사 품질을 높일 수 있다:

- **매출 Cutoff 테스트**: 기말 전후 거래의 귀속기간 적정성
- **환율 적용 적정성**: 외화매출의 KRW 환산 시 적용환율
- **대손·반품·할인 확인**: Credit Note, 반품, 할인 적용 여부
- **Incoterms 불일치 질의**: 엑셀 vs Invoice 운임조건 상이 건 회사 질의
- **선적서류 미제출 건 추가 요청**: Fail 건에 대한 증빙 보완 요청
- **완전성(Completeness) 테스트**: 출하기록 → GL 역추적

## 의존성

- Python 패키지: `pdfplumber`, `openpyxl`
- 설치: `pip install pdfplumber openpyxl --break-system-packages`

## 스크립트

핵심 Python 코드는 `scripts/` 디렉토리에 번들되어 있다:
- `scripts/tod_engine.py`: 메인 TOD 엔진 (증빙 분류, 텍스트 추출, 6개 테스트 수행)
- `scripts/tod_workpaper.py`: 엑셀 워크페이퍼 생성기

이 스크립트들은 범용 템플릿이다. 클라이언트별로 엑셀 구조(시트명, 행/열 범위)와 폴더 구조가 다를 수 있으므로, Phase 1에서 파악한 구조에 맞게 `samples` 추출 로직을 수정하여 사용한다.
